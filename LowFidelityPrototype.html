Design Concepts

During the brainstorming we listed three major components of a bill splitting service. The Login/Registration section, the Individual Bill View/Creation section, and the List of all Bill section. We decided to treat these three sections as three individual Low-Fidelity prototypes with the following Usability goals.

Login/Registration

  1. Remember Me
  2. Password Reset
  3. E-mail as Username
  4. Basic contact information
  5. Social media registration
  6. Optional profile picture
  7. Ability to register without payment method
  8. Option to register multiple payment methods
  9. Ability to edit profile information
  10. Ability to delete account
  11. Link to access profile from any page in website

Individual Bill view/Bill creation

  1. Bill Information
    • Cost
    • Date of Creation
    • Member Contribution
    • Preferred Payment
    • Text Description
    • Category/group
  2. Add proof of bill
  3. Add payers
  4. Submit bills
  5. Catagorize expenses into subgroups
  6. Add descriptions to expenses/subgroups
  7. Specify the contribution of each payer
  8. Indicate if someone has paid
  9. Comment section
  10. Make the bill easily shareable and with a perma-link

List of Bills

  1. Create, modify, delete, hide a Bill
  2. Sort bills by catagory
  3. Search function
  4. Option to pay quickly from the website

Separating the Bill splitting service into three parts allowed us to focus on the main actions the user will need to do. Using the 10 + 10 method we came up with the sketches you see below:

From these sketches we have found three themes. The Bubble design which organizes information in bubbles that can be moved around by gesture. The Card design which organizes bills as a list of tabs as one would sort a file drawer. And finally the Spreadsheet design which mimicks the popular spreadsheet template to organize the bills and expenses in those bills in tables. Then, made 3 pencil and paper prototypes that would embody these themes.

Prototype 3 - The Spreadsheet Design

The Spreadsheet design was inspired by traditional spreadsheets and Excel. The information is laid out methodically and precisely. It may be difficult to visualize the elements properly. This is a very cartisian solution.

1 / 5
Users log in by either entering their e-mail and password or through their social media account. If the user does not have an account they can create one.
2 / 5
Users must enter Full Name and email. User may connect a payment option to their account. User may upload a profile picture.
3 / 5
User can see the list of bills in a spreadsheet. Bills are arranged in terms of Name, Contribution, Ammount Due, Bill Creator and Date Created. User can sort these in Ascending or Descending order. User can search for a bill by name. User can navigate across additional pages of the list. User can create a new Bill.
4 / 5
User can see the list of expenses in a spreadsheet. Expenses are arranged in terms of Name, Ammount Due, Ammount Left to Pay,Payer and Date of Last Payment. User can sort these in Ascending or Descending order. User can select an expense to pay it.
5 / 5
User can see Name of the Bill being paid, the Name of the Payer, the total Ammount due of that payer, the Ammount Left to Pay of that payer and Date of Last Payment. User can selected a highlated payment option. User can enter an ammount to be paid or select a box to pay the remainder.
